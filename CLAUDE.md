# CLAUDE.md

## Project

Surge — agent-agnostic autonomous coding orchestrator. Pure Rust. Uses ACP (Agent Client Protocol) to connect to any coding agent (Claude Code, Copilot CLI, Zed Agent).

## Commands

```
cargo build                    # build all crates
cargo test                     # run all tests
cargo run -p surge-cli         # run CLI
cargo run -p surge-cli -- init --default
cargo run -p surge-cli -- project describe
cargo run -p surge-cli -- doctor matrix            # show sandbox delegation matrix
cargo run -p surge-cli -- doctor report --format json
cargo clippy --workspace       # lint
```

## Architecture

Multi-crate workspace. See `docs/ARCHITECTURE.md` for vision and design notes (in progress).

- `surge-core` — shared types: SpecId, TaskId, TaskState FSM, SurgeConfig, AgentConfig
- `surge-acp` — ACP Client trait implementation, AgentPool, AgentConnection, event system
- `surge-cli` — clap-based CLI: `surge init`, `surge project describe`, `surge bootstrap`, `surge engine`, `surge feature`, `surge migrate-spec`
- `surge-orchestrator::project_context` — deterministic project scan, `project.md` generation, and run-level `project_context` seed helper

## Key Design Decisions

- **Agent transport: ACP only.** No direct subprocess hacks or per-CLI stdout parsers. Rationale and accepted costs in [ADR-0006](docs/adr/0006-acp-only-transport.md).
- **Sandbox is delegated.** Surge configures the agent runtime's native sandbox via the matrix in [`docs/sandbox-matrix.md`](docs/sandbox-matrix.md); elevation lifecycle and operator approval flow live in [`docs/elevation-runbook.md`](docs/elevation-runbook.md). Inspect with `surge doctor matrix` / `surge doctor report`.
- `surge init --default` and interactive `surge init` are the project onboarding paths.
- `project.md` is generated by `surge project describe` and captured into new runs at start time; do not read it lazily during stage execution.
- Flow graphs use TOML format (not markdown) for type safety and git-friendly diffs; see [`docs/conventions/flow.md`](docs/conventions/flow.md).
- Every task runs in an isolated git worktree via `git2` crate.
- State machine for tasks: Draft → Planning → Planned → Executing → QaReview → HumanReview → Merging → Completed
- IDs use ULID (`ulid` crate).
- Async runtime: `tokio`.
- Error handling: `thiserror` for library crates, `anyhow` for CLI.

## Coding Standards

- Rust 2024 edition, stable toolchain
- `#[must_use]` on functions returning Result
- Public API documented with `///` doc comments
- No `unwrap()` in library code — use `?` or explicit error handling
- Tests next to code in `#[cfg(test)]` modules
