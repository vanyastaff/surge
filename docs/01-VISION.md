# Surge — Autonomous Coding Orchestrator

> **Any Agent. One Protocol. Pure Rust.**

---

## Название

**Surge** (англ. *волна, всплеск, мощный поток*) — олицетворяет мощный рывок в автономной разработке. Surge — это волна продуктивности, которая подхватывает задачу и превращает спецификацию в готовый код, используя любого AI-агента через единый протокол.

Альтернативные варианты для обсуждения:
- **Ferrum** — латынь "железо", прямая связь с Rust
- **Iskra** — "искра", начало автономного кодинга
- **Strato** — стратосфера, высокоуровневая оркестрация

---

## Миссия

Surge — первый **agent-agnostic** автономный оркестратор для разработки, написанный полностью на Rust. Один бинарник, один протокол (ACP), любой AI-агент — Claude Code, GitHub Copilot, Zed, или любой будущий ACP-совместимый агент.

## Проблема

Современные инструменты автономного кодинга (Aperant/Auto-Claude, Cursor Background Agents) привязаны к одному провайдеру:

1. **Вендор-зависимость** — Aperant работает только с Claude Code, Cursor только с собственной моделью
2. **Хрупкость** — Electron + Python + Node.js — три рантайма, сотни зависимостей, частые краши
3. **Отсутствие стандарта** — каждый инструмент изобретает свой протокол общения с агентами
4. **Ограничения подписки** — OAuth-токены привязаны к конкретным продуктам, нельзя легально использовать в сторонних инструментах

## Решение

Surge строится на трёх принципах:

### 1. ACP-First (Agent Client Protocol)

Surge реализует `Client` trait из ACP — стандартного протокола от JetBrains и Zed Industries. Это значит:
- Подключай **любого** ACP-совместимого агента
- Claude Code, Copilot CLI, Zed Agent — все работают через один интерфейс
- Будущие агенты совместимы автоматически
- Возможно использовать **разных агентов для разных подзадач** в рамках одного spec'а

### 2. Pure Rust — Единый Бинарник

Никаких Electron, Python, Node.js. Один `surge` бинарник:
- Мгновенный запуск (< 50ms)
- Потребление памяти ~30MB vs ~500MB+ у Aperant
- Нет зависимостей — скачал и работай
- Нативная кроссплатформенность (Windows, macOS, Linux)

### 3. Spec-Driven Development

Формализованный пайплайн превращения задачи в код:
```
Задача → Spec → Plan → Subtasks → [Agent Execution] → QA → Review → Merge
```

## Ключевые отличия от Aperant

| Аспект | Aperant (Auto-Claude) | Surge |
|--------|----------------------|-------|
| **Агенты** | Только Claude Code | Любой ACP-агент |
| **Стек** | Electron + Python + Node.js | Pure Rust, один бинарник |
| **Размер** | ~300MB+ установки | ~15MB бинарник |
| **Запуск** | 5-10 секунд | < 50ms |
| **RAM** | 500MB+ | ~30MB |
| **Протокол** | Проприетарный subprocess | ACP (открытый стандарт) |
| **Мульти-агент** | Нет | Claude для планирования, Copilot для кодинга |
| **Файловый формат** | Разрозненные MD-файлы | Структурированный TOML/JSON spec |
| **Worktrees** | Базовая поддержка | Продвинутая с автомёрджем |
| **Лицензия** | AGPL-3.0 | MIT / Apache-2.0 |

## Целевая аудитория

1. **Разработчики, использующие Claude Code / Copilot CLI** — хотят автоматизацию без привязки к конкретному IDE
2. **Команды** — spec-driven workflow с code review и QA
3. **Open-source контрибьюторы** — MIT-лицензия, Rust экосистема
4. **Пользователи Aperant** — ищут стабильную альтернативу без зависимости от Electron

## Философия

- **Агент — расходный материал.** Surge спроектирован так, чтобы замена агента не требовала изменений в workflow.
- **Spec — источник истины.** Спецификация живёт дольше любой сессии агента. Она воспроизводима, версионируема, ревьюабельна.
- **Rust — не ограничение, а преимущество.** Type safety, fearless concurrency, zero-cost abstractions — это не академические аргументы, а практическое качество продукта.
