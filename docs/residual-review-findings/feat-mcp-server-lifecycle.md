## Residual Review Findings

- **#1 [P1 ¬∑ gated_auto] `crates/surge-mcp/src/redact.rs:111` ‚Äî Secret redaction misses connection-string URL credentials.** `postgres://user:pass@host`, `redis://`, `http://user:pass@proxy`, and `DATABASE_URL=`/`DSN=`/`connection_string=` values are leaked verbatim to the `mcp::child::stderr` tracing target and the on-disk stderr file. (security, confidence 100) ‚Äî *Fix:* add a userinfo-masking pass (mask between `://` and `@`) and extend `SENSITIVE_KEYS` with `database_url`/`db_url`/`dsn`/`connection_string`/`redis_url`/`amqp_url`/`uri`; add regression tests for credential URLs.
- **#2 [P1 ¬∑ gated_auto] `crates/surge-daemon/src/server.rs:842` ‚Äî `McpProbe` is unbounded and serialized with the IPC connection.** One slow or hung MCP server stalls the entire daemon connection ‚Äî including the operator's own `StopRun`/`Shutdown` on that connection. (reliability + adversarial, confidence 75) ‚Äî *Fix:* wrap each per-server probe in a bounded timeout independent of `call_timeout` (~10‚Äì15s) and/or run the per-server loop concurrently (`JoinSet`) so total latency is `max()` not `sum()`; or dispatch `McpProbe` on a spawned task so the read loop is not blocked.
- **#3 [P2 ¬∑ gated_auto] `crates/surge-mcp/src/redact.rs:96` ‚Äî Short (<32 char) secrets leak when key and value are whitespace-separated.** `Authorization:Bearer <shortTok>` (no space) and pretty-printed JSON layouts evade the value mask. (security, confidence 100) ‚Äî *Fix:* when the prior stripped token is a sensitive key (or a bare `:`/`=` immediately after one), set `redact_next_token` so the next value token is masked regardless of length; add tests for spaced JSON and short-token forms.
- **#4 [P2 ¬∑ gated_auto] `crates/surge-daemon/src/server.rs:120` ‚Äî Daemon socket `0600` is applied after bind, not at bind.** TOCTOU window where another local user can reach the unauthenticated IPC socket; contradicts ADR-0014 decision 6 ("mode `0600` at bind"). (security, confidence 75) ‚Äî *Fix:* create the `~/.surge` daemon dir with mode `0700` (`DirBuilderExt`) so the brief socket-mode window is unreachable; reconcile ADR-0014 wording with the implementation.
- **#5 [P2 ¬∑ gated_auto] `crates/surge-daemon/src/server.rs:847` ‚Äî `McpProbe` swallows config-discovery errors.** `SurgeConfig::discover().unwrap_or_default()` turns a malformed/unreadable `surge.toml` into `McpProbeOk { servers: [] }`, indistinguishable from "no servers configured". (reliability + kieran-rust, confidence 80) ‚Äî *Fix:* `match SurgeConfig::discover()`; on `Err` return `DaemonResponse::Error`. `discover_from` already returns `Ok(default)` when no `surge.toml` exists, so only a genuine OS failure reaches `Err`.
- **#6 [P2 ¬∑ manual] `crates/surge-mcp/src/connection.rs` ‚Äî Monitor-detected `RestartExhausted` produces no `EscalationRequested`.** `spawn_health_monitor`'s `let _ = me.ensure_connected()` discards the error; escalations are only drained on the agent `dispatch()` path. A server that dies between tool calls (monitor-detected) on a run that ends before the next MCP dispatch gives the AFK operator no cockpit card ‚Äî a partial gap in the U3/A2 guarantee. (maintainability + testing, confidence ~70) ‚Äî *Fix:* give the health monitor a path to surface exhaustion (a connection-level exhausted flag the engine drains on teardown, or an mpsc the orchestrator forwards to `EscalationRequested`) so monitor-only failures still escalate.
- **#7 [P2 ¬∑ gated_auto] `crates/surge-mcp/src/connection.rs:676` ‚Äî `stderr_forwarder` rewrites the whole bounded file per stderr line.** O(n¬≤) disk churn for a chatty MCP server (rebuild VecDeque‚ÜíString‚Üífull `fs::write` every line). (performance + adversarial, confidence 75) ‚Äî *Fix:* debounce/throttle the rewrite (flush every N ms or K lines) or append-with-periodic-rotation; keep the bounded-tail invariant (do not trade it for an unbounded file).
- **#8 [P2 ¬∑ manual] `crates/surge-orchestrator/src/engine/tools/routing.rs:170` ‚Äî No unit test exercises the real `RestartExhausted ‚Üí drain_mcp_escalations ‚Üí EscalationRequested` path.** The `Mutex` push site is never reached by a test producing a real `McpError::RestartExhausted`; `engine_mcp_terminal_shutdown.rs` only covers registry teardown. (testing + maintainability, confidence 80) ‚Äî *Fix:* add a `tokio::time::pause`/`advance` routing.rs test that drives a connection to `Exhausted`, dispatches a tool to it, and asserts `drain_mcp_escalations()` returns one `McpEscalation` then a second drain is empty.
- **#9 [P3 ¬∑ manual] `crates/surge-mcp/src/connection.rs:545` ‚Äî U11 unhealthy-after-N-failures threshold is not deterministically tested.** Only the cancel seam is covered (`health_monitor_exits_promptly_on_token_cancel`). (testing, confidence 75) ‚Äî *Fix:* add a `tokio::time::pause` test advancing through `HEALTH_FAIL_THRESHOLD` ticks against a closed/failing service handle and asserting `status() == McpHealth::Unhealthy`.
